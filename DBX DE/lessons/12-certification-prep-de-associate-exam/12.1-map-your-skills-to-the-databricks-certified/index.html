The 7 domains — weight, coverage & an exam-style snippet

1 · Databricks Intelligence Platform · 6% · Stage 1

Lakehouse, workspace/architecture, medallion. Lowest weight — know the concepts.

-- Recognize the 3-level Unity Catalog namespace (a platform fundamental):
SELECT * FROM main.sales.orders;   -- catalog.schema.table

2 · Data Ingestion and Loading · 21% · Stage 4

Auto Loader (cloudFiles), COPY INTO, Lakeflow Connect, schema evolution.

# Auto Loader — incremental file ingestion (high-yield):
(spark.readStream.format("cloudFiles")
   .option("cloudFiles.format", "json")
   .option("cloudFiles.schemaLocation", "/Volumes/main/raw/_schema")
   .load("/Volumes/main/raw/orders")
   .writeStream.option("checkpointLocation", "/Volumes/main/raw/_ckpt")
   .toTable("main.bronze.orders"))

3 · Data Transformation and Modeling · 22% · Stages 2, 5, 6

Highest weight. Delta, MERGE, SDP, expectations, PySpark/Spark-SQL ETL.

-- MERGE (upsert) — a classic exam item:
MERGE INTO main.silver.customers t
USING updates s ON t.id = s.id
WHEN MATCHED THEN UPDATE SET *
WHEN NOT MATCHED THEN INSERT *;

4 · Working with Lakeflow Jobs · 16% · Stage 7

Tasks & dependencies, scheduling with Quartz cron, retries, alerts.

# 6-field Quartz cron (NOT 5-field Unix) — frequently tested:
schedule: { quartz_cron_expression: "0 0 2 * * ?", timezone_id: "UTC" }  # 02:00 daily

5 · Implementing CI/CD · 10% · Stage 11

Databricks Asset Bundles, Git folders, dev/prod targets, deploy flow.

databricks bundle validate          # check config (CI on PRs)
databricks bundle deploy -t prod    # promote on merge

6 · Troubleshooting, Monitoring & Optimization · 10% · Stages 3, 7, 10

OPTIMIZE/ZORDER/Liquid Clustering, VACUUM, query profile, run monitoring.

-- Liquid Clustering — added to the current syllabus (know it):
ALTER TABLE main.silver.events CLUSTER BY (event_date, region);
OPTIMIZE main.silver.events;

7 · Governance and Security · 15% · Stages 8, 9

Unity Catalog grants, row filters / column masks, ABAC, Delta Sharing.

-- A GRANT on the UC 3-level namespace (high-yield governance item):
GRANT SELECT ON TABLE main.sales.orders TO `analysts`;

Logistics (verified — re-check before booking)

  • 45 scored questions · 90 minutes · multiple choice · proctored (online or test center).
  • Fee $200 · no prerequisites (hands-on recommended) · valid 2 years.
  • Passing grade ≈ 70% (~32/45) — ⚠️ verify; the official page doesn't publish the exact cut score.
  • Languages: EN, JA, PT-BR, KO · no test aids · latest guide reflects Lakeflow naming + CI/CD + PySpark ETL + Liquid Clustering.

Why it matters

  • Study to the weights — Transformation + Ingestion + Jobs ≈ 59%; don't over-invest in the 6% Platform domain.
  • Mapping domains → stages turns "study everything" into a targeted plan.
🗓️ Real-world use case: two weeks out — most time on Stages 2/4/5 (Delta + ingestion + pipelines), a solid block on Stage 8 (governance, 15%), a quick review of Stage 1 (6%).

Uses, edge cases & limitations

  • Uses: a weighted study plan, a coverage checklist, deciding what to review last.
  • Edge cases: domain names/weights change between exam versions — re-check the official guide near your date; recent updates added Liquid Clustering + Lakeflow naming.
  • Limitation: weights guide time, not depth (a 6% domain can still have a must-get question); the exam tests PySpark/Spark-SQL ETL (outside this plan) — review separately.

Common gotchas

  • ❌ Studying every topic equally — weight time to Ingestion + Transformation + Jobs.
  • ❌ Trusting old weightings — verify the current exam guide.
  • ❌ Skipping PySpark/Spark-SQL ETL because this plan excludes Spark core — the exam tests it.
  • ❌ Ignoring the 15% Governance domain.
  • ❌ Writing 5-field Unix cron for Jobs — Databricks uses 6-field Quartz.
