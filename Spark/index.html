PySpark Performance
& Spark internals.

Eleven connected lessons on how Apache Spark actually runs your PySpark — told as one story: your code becomes jobs, stages, and tasks across a driver and executors; joins and wide operations trigger the expensive shuffle; memory decides what runs, spills, or OOMs; and every tuning lever (AQE, caching, pruning, salting, GC, bucketing) bends that execution. Each lesson is a self-contained interactive page with a runnable notebook and a markdown companion.

Then, in order: read less (partition pruning / DPP, column pruning) → avoid or repair the shuffle (broadcast the small side, bucket repeated joins, let AQE coalesce) → fix skew (AQE skew join, then salting) → reuse work (cache only reused DataFrames) → size memory & GC last.
